Conclusions

In general I found Backbone.js to be the easiest to work with, followed by Knockout.js. But, Backbone.js is also the least powerful of the libraries. Both Knockout.js and Ember.js provide more features. There's an obvious tradeoff between having more features and a steeper learning curve.

Knockout.js required the least amount of code to get this use case up and running but this might be just because it's uniquely suited to this task. It also has the best tutorial system.

The toughest for me was Ember.js. Like WPF, it seems like a very powerful framework that would be very useful if one found the time to master all its intricacies but I'd almost say it was too difficult to use. It felt too abstracted and sophisticated. Then again, this might be unfair since I didn't spend too long on learning these libraries.
